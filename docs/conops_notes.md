# ConOps Notes

The scenarios in this repo represent notional disaster-response mission threads.

Each scenario is broken into:

- a mission objective
- ordered operational phases
- phase durations
- active systems or teams

This makes it possible to answer:

- which systems are active together
- where major handoffs happen
- how long a mission thread lasts

The repo is intentionally lightweight and reviewer-friendly, but structured enough to demonstrate operational reasoning in interviews.
